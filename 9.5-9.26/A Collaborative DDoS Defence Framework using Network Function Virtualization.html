A Collaborative DDoS Defence Framework using Network Function Virtualization

security

Rashidi B , Fung C , Bertino E . A Collaborative DDoS Defence Framework using Network Function Virtualization[J]. IEEE Transactions on Information Forensics and Security, 2017:1-1.

本文利用NVF提出CoFence,使不同的域网通过资源共享互相帮助来解决攻击，并设计动态资源分配机制以提高分配效率。（本文重点是提出了一个资源协作方案）

基本介绍

在NVF中网络工能以软件的形式提供和实现，并运行在商业硬件上，该方式有更好的灵活性和更低的部署代价，NVF也更好地支持服务升级。本文基于NFV提出更优的资源管理分配方案使多个域合作解决DDOS攻击。

Collaborative DDoS Defense

CoFence(CF)的目的是使不同域网互相帮助以抑制DDOS。在CF网中每个域有一个网关一虚拟ISP，虚拟ISP的目的便是检测和过滤DDOS攻击。在NFV中ISP可被创建并依需分配资源，域加入CF时可选择是不共享ISP和配置最大愿意处理的流量。图2为CF的一个说是，当有攻击时，攻击流量超过本地ISP容量便将其导入其他ISP，公被过滤的流量会被交付给victim。

图3为网络资源协作说明

所有被共享的ISP资源都在虚拟网中被组织，域与依靠的邻域连接，每个域都要维护邻居信息，如贡献点，邻居所需要的资源，可提供给邻居的资源。
图4为域公共服务器内部，

每个虚拟IPS需要的功能有：

  • 交流组件需要1.请求帮助和提供帮助；2.请求添加邻居和响邻居添加请求；3.请求移除邻居和向应邻居移除请求。图5为向域3请求帮助的说明。
  • 资源分配：接收到帮助需要后就要分配资源需及时作出最优的资源分配决策，该部分是本文重点。
