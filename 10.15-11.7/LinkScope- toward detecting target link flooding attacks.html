security

这一篇为将上一篇整理后发的会议，略有改变。

架构

  1. 拓扑分析，找potential link和有这些link的end-to-end的path，
    拓扑分析机制使用的[26]
  2. 使用path measurement 模块进行跳到跳，端到端测量
  3. 攻击检测模块将数据转成特征向量，再进行异常检测。
  4. target link 定位模块对受攻击的link进行定位。

过程

整个过程如图1。1.找到可能的link和path;2.对选择的path作测量；3.4.对结果构建特征向量并检测异常；5.有异常就定位。

  • target Link Monitoring
    用每个网络的前30的link作target，当有LFA时找其path中被攻击的link的后面的link，结果如表3,可见超过80%的link可被找到。

  • Evaluating the attack detection Module
    LinkScope是否可以有效探索被firewall和IDS保护的seb server?，本文运行linkscope 来探索9个web site，一天。使用谷歌4个地区的VM分别probe6个server，收集到24个结果，评估FP（假设无LFA），每个path探索一次180秒，持续10天，利用第4天和第10天的结果算FP，如表4,u,v为前面说过的阈值。结果表明u=2,v-=2时FP小于25%。Pa为同时出现异常的path的个数，当Pa=2时FP小于0.3%。

    与其他检测算法相比：本文算法的FP最低。基于多个path的检测能降低FP。
