Software Defined Traffic Measurement with OpenSketch

本文提供了一个流量测量工程框架OpenSketch(OS)，兼顾通用性和有效性。

Introduction

本文提出OS将数据平面与控制平面分离，数据平面有一个3步管道：hashing,filtering,counting;控制平面提供测量库为不同任务分配不同资源并配置管道。
网络管理可分为测量和控制，本文注重测量，目前的测量方法要么缺乏普适性，要么缺乏有效性，本文受OF启发，也将分为控制平面和数据平面。本文提出重设S处的API使其更加通用和有效，并在控制器处编程更加简便。本文提到的控制器平SDN里面的控制器并不是一个东西，本文中所言控制平面与数据平面也都是OS中的，于SDN并无关系。代码开源见【5】。在此

Design

OpenSketch Data Plane

OS数据平面设计通用性支持各种测量任务，有效，节省TCAM,SRAM,只需要少量的hash函数。

Generic and efficient data plane

由两个功能组成：选择测量包；存储/导出测量数据；

Picking the packets to measure

利用hash和分类功能的结合来表示需要测量流的体集，如：Servers set，可以用bloom filter压缩表示。分类也可以用于集中处理特征流，如特征列表server或租户。例职：192.168.1.1的流量太多，可以将其单独从192.168.1.1/32中过滤出来。使用一个计数器计数该IP流量，另一个计数器计数192.168.1.0/24的流量，流分类可以使用通配符进行，允许某些位设为 don't care，如：可将192.168.1.0/24的低8位设为 don't care。

Storing and exporting the data

OS使用复杂索引表，每个表项包含计数器而不包含流的域，计数器可以被 关联到不同实体，如：microflow,wildcard flow,hash value。OS可以很容易将表导到时控制器，这种方式使OS用更少的内存实现更灵活的数据收集。复杂索收要用到hash和分组模块，可利用商用S的组件计算这些索引。对于需要识别特定流的流量任务，控制器可以利用基于分类的索引或逆向分析hash值以获得索引的办法来维护计数器和映射。

OpenSketch data plane

有三个stage组成，结构如图2

hash是用来降解测量数据的，分类是用来选择流，计数用来累计流统计。例如对子网192.168.1.0/24统计入流源IP,hash利用包源IP域计算hash值；分类：匹配目的IP规则（dst:192.168.1.0/241）.每个rule都有一个索引域被用来统计，本例中索引为1则计数，缺省为-1，则不计数；计数功能进行计数和统计。

Build on eisting switch components

在商用S上实现OS的data plane

  • 需要实现一些简单的hash函数
    hash不同，sketch对hash的要求也不是，但经验表明4-8 three-wise or five-wise independent hash function已够满足许多测量需求，并可以硬件上高效实现。
  • 需要分配少量TCAM以进行分类
  • 计数实现在SRAM中
    计数器存在SRAM中，由于不同sketch需要不同大小的计数器，本文实殃a list of logical tables，这些表可以在相同或不同sketch中代表不同计数器。

OS控制器提供一个实现好的sketch库和两个组件，管理器：自动以最好memory-accuracy tradeoff配置sketch；资源分配器：划分S的资源给多个测量任务。

protype implementation

图4为OS架构

* 数据平面
在FPGA中实现以测试。因为不改变packet bus，因此无延迟，不影响吞吐。来包后使用header parser获得包头，hash来计算机hash值，再发给wildcard lookup匹配，若匹配，更新计数。
* 控制器
用C++实现7个sketch:bitmap,PCSA,hash table,multi-resolution classifier,bloom filter,count-min,seversible sketch。每个sketch有两个功能：配置参数，查询统计。并实现管理器和资源分配器，测量程序可以定期从sketch查结果，并依此插新sketch或更改次确率需求。报告统计结果后data plane 计数器置0,历史统计信息在控制器中维护。
