On the Feasibility of Rerouting-Based DDoS Defenses

security

M. Tran, M. S. Kang, H. Hsiao, W. Chiang, S. Tung and Y. Wang, "," 2019 IEEE Symposium on Security and Privacy (SP), San Francisco, CA, USA, 2019, pp. 1169-1184.

本文对RAC（Routing Around Congestion:Defeating DDoS Attacks and Adverse Network Condition via Reactive BGP Routing,S&P）防御DDOS方法进行了深入研究并对在实际应用上的可行性进行了探讨最终得到结论:RAC防御方法在当前网络环境下并不可行。

introduction

DDOS攻击当前流行的一个种类为transit-link flooding，即攻击某个link使这个链路上所有的流产生丢包，由于攻击不发生在终端因此很难被检测与御防。已有的解决办法前提是所有的ISP能相互合作而这个在现实中无法实现。最近有个人提出了RAC方法，发在S&P上，它不需要对当前网络作任何更改因为它只依赖于BGP路由控制。本文对该方法进行深入的可行性分析发现该方法并不能解决洪泛攻击一个有效的RAC应保证：1.一定可以建立detour path；2.所建立的detour path能与非重要流隔离，即只让重要流使用。这两个条件在当前网络环境中只能实现一个。本文主要目标为列出一些server/link洪泛攻击的结论。改管BGP消息理论上支持2034个AS长度的path，但实际上思科路由器默认丢弃超过256长度的BGP消息。

  • Transic-Link Flooding Attacks

    攻击特点：非直接攻击，终端的IDS无法检测；无法与正常流区分，造成阻塞使流被丢弃而产生的额外效果为攻击目标。

RAC防御的两个重要属性： 1.path isolation,非重要流不走这条路。2.Guranteed detour establishment,保证一定要找到链路。
RAC的目的在于是被attack时及早找到一个额外path给重要流使用。

这两个属性是否可共存的讨论

首先分析path isolation需求和BGP poison模式，通过研究BGP UPDATE消息调研获得高置信度detour path所需条件。

保证detour path建立的条件分析

为保证建立detour path，D发的BGP poison消息必须被D-C上所有AS收到，并不被过滤，本节探索AS如何处理长AS path的BGP UPDATE消息。

  • Active measurement
    想通过AS发具有超长path的poison消息是不可行的。因为1.router不允许配置2.机构担心这会破坏他们的路由器。

  • Passive measurement
    收集6个月内大量的UPDATE消息并统计，分析了370亿个UPDATE消息可见大多数长消息都是AS预置消息，即有 多重复的AS号。这种长的UPDATG消息一般会被忽视，图9为消息中path长度的分布，对每个path长度列出消息个数，对于30+path长度只占0.01%.

    由图9可见：a.长度为[30,75)的长消息数大大减少；b.之后的[75,255）消息数并未下降；c.长度超过255的消息数急剧下降。
    推测原因为：A。一些AS会过滤掉长度为[30,75)的消息；B。AS都不过滤长度为[75,255）的消息；C。大多数AS都过滤掉长度超过255的消息。
    本文确定该推测方法准确，原因如下：

    1. 供应商鼓励过滤40+长度的消息与A相合。
    2. 未发现任何路由会过滤长度为75-255的消息与B相合。
    3. 思科路由会丢弃255+长度的消息，思科路由是行业的主流，符合c和C。

  • Filtering practices in ISPs:当前ISP也会丢弃长path的消息，这些事实与以上推测吻合。

两个属性相互矛盾

要想实现路径隔离就要使UPDATE消息有超过255个AS，而超过了就被过滤，这样便无法保证detour path的建立。

NON-isolated Detour path and path leakage

保证隔离性便无法保证path被建立，本节讨论不保证隔离性时出现的问题。

Metric to evaluate Non-isolated detour paths

AS间链路不公开，因此额外path的可用带宽也无法估计。但可通过非额外path的poison的AS来因为非重要AS收到poison消息后会使非重要流经过detour path，并使用path leakage作为detour path可用带宽的指标。其中，path leakage：AS不属于detour path，但却收到了GBP UPDATE消息，称AS观察到了path leakage，指标为：观察到path leakage的个数，即非detour path外，收到BGP UPDATE的AS个数。图10为举例说明，D要发BGP UPDATE消息给CXYZ，但EFG却收到了。

Minimization of Path leakage due to Non-isolation detours

首先定义问题：找非隔离额外path，使其path leakage最小，再计算近似最优解。
定义P1：D创建UPDATE消息U，并在额外path 上发送。
目标：找到AS集P，使U的path leakage最小，并且,B为最大可允许的P的尺寸。
该问题是典型的MinSBCC问题[28]，详见附录B，是NPC问题。简化P1并用贪心算法求近似解：1.从detour path的AS的邻居中找P而不是全网AS中找；2.定义Q为可poison的AS，限制Q中AS传递U只能传给已poison的AS以限制path leakage.而这需要被poison的AS记为isolating set .算法目标为找到不在detour path上的最大AS数，并且他们的并集元素数小于B，每轮找最小的并将加到P，最终输出需要被poison的P，详见附录C。

Adaptive link flooding attack versus adaptive RAC defense

攻击对抗RAC防御
当有path leakage产生时，攻击者可以实时学正能量detour path 并阻塞它，将该攻击称为：detour-learning attack。而RAC对这种攻击抵御较差。

Detour-learning attack based on path leakage

  • 攻击模型
    1. 攻击目标：及时找到额外path并将其阻塞。
    2. 攻击者的能力：假设攻击者无法从BGP路由表中获取detour path，且没有bot在关键AS内。
    3. knowledge of the existence of RAC defense:攻击者可以通过公开BGP记录库中找到C和D上部署了RAC防御。
  • learning real-time detour path changes
    攻击者可以通过botnet的path测量来实时找到新的detour path，攻击者每秒测一次AS到目的AS的路由（该AS内有bot），通过traceroute就能检测到，大多数ISP允许tracetroue。因此当路由改变时攻击者可能利用bot很快检测到。若bot不在C（critical AS），则只能测到detour path的一部分，找到detour path后将其每个link作为目标link来攻击，如图12,若公选most popular link则不够准确（在图12中可见E-Y是most popular link，而实际上X-Y才是），这是因第每个bot只看到部分detour path，且它们都不一样。本文提出使用distance-based target-link selection 算法，倾向于更靠近C的AS测量的结果，如图12中I离C最近，则更倾向于I的结果。
  • evaluation of detour-learning attack
    使用Mirai[15]模型来模拟DDOS僵尸网络，因为[15]是一个有代表性的大僵尸网络。从Mirai-5K，Mirai-1K中来选网络来模拟。
    1. detection of detour path establishment:通过计数有bot的AS收到BGP poison消息来评估对detour path的检测，图13A表明对于含1000和5k的AS(其内有bot)检测到path leakage率分别为96%和99%
    2. accuracy of target-link selection algorithm：找到的目标Linkdqpud detour path上便认为成功找到，对于Mirai,Mirai-5k,Mirai-1k而言成功率分别为：94.1%，86.4%，79.2%.
    3. link-flooding attack using botnet dataset:确认选择的正确目标link是否被洪泛，图13B表明在Mirai,Mirai-5k,mirai-1k集中分别有94.69%,93.17%,90.4%已选的目标Link上有僵尸网络流。结认：攻击者可轻易地攻击找到的目标link。

slow reaction of RAC defense against adaptive attacks

由于重新发现一条detour path需要undo之前做的改变，这个过程由于leakage原因所以慢，因此攻击敏捷，防御迟钝，使该防御方法无效。

  • path hunting
    path hunting问题，如图14解释，T0,T1 A发自己的BGP UPDATE消息，T2回收，而T3时屾于C的最近路由（CA）被回收，所以传播次近路由（BA）给D，而次近路由又被A回收，此迭代过程直到T5结束，花了4个时间单位，大大长于发UPDATE的时间，称为path hunting问题。因此找新的detour path会传很久的回收消息。
  • evaluation of path hunting in RAC
    使用[13]SSFNet和[2]BRITE来生成网络拓扑并以此估计path hunting 的时是。每次模拟随机选择一个AS并让其发GBP poison消息，再回收，并计算时间。（路由表到达稳定则回收完成。）
    图15为随机结果图，1000AS时收敛时间为85秒而更新时间平均6秒，这拓扑比现实小得多，因此RAC确实无法抵御攻击。使用不同的RFD[54]和MRAI[49]配置，因定500个AS再测结果如图16.

How to make the RAC defense possible

若要使RAC呆用，需要满足1.允许BGP协议中UPDATE消息有更长的AS path,超过2034[52]；2.所有的BGP路由，不管其AS path长度多少都接受GBP UPDATE消息。当前这两个条件根本无法被满，满足了也会被恶意利用。

  • 结论：基于重路由的DDOS防御当前很难实现，因为：域内路由协议没有更细粒度的支持；2.路由特征会被恶意利用。
