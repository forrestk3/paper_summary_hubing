DDoS attack detection under SDN context

security

Xu Y, Liu Y. DDoS attack detection under SDN context[C]//IEEE INFOCOM 2016-The 35th Annual IEEE International Conference on Computer Communications. IEEE, 2016: 1-9.

本文利用SDN的流控制和计数能力来检测DDOS攻击。通过对可疑IP range插入rule以捕获入和出的流量；对S插规则时监控以协调利用有限的TCAM。

System overview

DDoS检测由两部分组成：victim-detection,post-detection(攻击者检测).目标为快速准确地测victim IP，对可能的victim IP，利用rule记录in ip和out ip的流量。由于TCAM有限，先对一个大范围IP检测，若有攻击特性再细测以找到最精确的IP范围。这两部分可以sequentially或simultaneously实现，以下分别讲解两种方法及优劣。

A.Victim dection

1. initial rule placement

当对DDOS攻击无先验知识时就需要监控所有的IP，我们的目标就是要使监控所的范围更小以获得更细粒度的rule来监控相应IP的in IP 流量和out IP 流量。每个IP range用一个前缀表示，对于源IP range使用，目的IP range使用,我们想要通过所有相同S序列的path和其相应的反转path的两个IP range对.假设为：每一个IP range连的是PoP 路由，即：两个PoP路由的流量都通过所有相同的S。如果PoP后的IP不能用一个前缀表示，则细分到多个前缀表示；若某相IP range不满足则划分PoP的IP直到每个pair内的主机能通过相同的S序列，最后将每个pair 称为一个flow。（这一部分就是将全网划分为来回流量都同S序列的pair的.）找到flow后就为每个f下发一个rule。

3. Detection Rule Adaptation

对有疑似被攻击的IP range挺更细的规则，无的则用精粒度规则代替，方法称为：spatial adapation[13].A2中说明adaption Algorithm，对每一个目的IP range d，使用看它是否，若它和它的sibling range d:sib(d)都是false，则可用common parent Parent(d)替换它们的rule，若,则更细粒度rule检查。粒度到了极限就做攻击者检测，每次进行粒度改变都要使用A1。
