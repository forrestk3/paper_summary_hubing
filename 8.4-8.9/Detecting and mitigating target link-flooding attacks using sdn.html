Detecting and mitigating target link-flooding attacks using sdn

security

Wang J, Wen R, Li J, et al.[J]. IEEE Transactions on Dependable and Secure Computing, 2018.

A类
本文提出LAFDefender，由于LFA是通过大量的bot来同步阻塞link的攻击方法，本文通过利用SDN的可编程性，全局网络视图，流的可追踪性来高效检测LFA（link flooding attack）并提出多路径重路由和追踪bot并阻塞的方法来缓解LFA。

Introduction and background

link flood attack

LFA中的Coremelt攻击步骤为：1.找目标link；2.找可在link上形成flow的host；3.host产生流量，使标link过载。
LFA中的Crossfire攻击步骤为：1.构建link map；2.计算流密度和选择标link；3.bot协调；4.轮流攻击。

System overview

整体视图见图1,由四个模块组成：target link selection,link congestion monitoring,traffic rerouting,malicious traffic blocking.

target link selection module:与C交互，获得S中的entry以分析flowpath及链路上的流密度,并标记高流密度的目标link。
link congestion monitoring module:获得前面的目标link，并在这些link上布置代理监控它的流量信息以报告给C，使C来决定link是否阻塞。
traffic rerouting:对阻塞link中的flow重路由从而暂时缓解。
Malicious Raffic Blocking:识别bot并阻塞恶意流。

flowpath的获得过程：1.获得header of the head node作为flowpath的开始结点，能过C得到下一跳S1；2.读S1流表中的header information并转换为二进制向量，并计算head node的目的IP与S1源IP的交，如果结果不空，则下一跳为S1；类似地，通过S1的目的IP与S1下一跳的源IP的交以获得S2,重复上述过程直到获得flowpath。

link congestion monitoring

link congestion monitoring module运行在C中，当收到目标link时向其装监控代理并实时监控数据，当目标link变成非目标link时移除代理。
判断link是否阻塞的指标：Pack loss rate > MLoss;RTT > MRTT;Available bandwidth < MABW。
这三个阈值可参考[16]来计算。
